## Table View Deep Dive

The table view is the most powerful for managing large job lists.

### Default Columns

Every job list shows these columns by default:

| Column | Description | Sortable? |
|--------|-------------|-----------|
| **Job #** | Unique job number (e.g., JOB-1234) | ✅ Yes |
| **Client** | Client name (click to open profile) | ✅ Yes |
| **Description** | What needs to be done | ❌ No |
| **Status** | Current lifecycle stage | ✅ Yes |
| **Assigned** | Tech(s) assigned to the job | ✅ Yes |
| **Date** | Next visit date (or created date if unscheduled) | ✅ Yes |
| **Value** | Total job value (sum of items) | ✅ Yes |
| **Actions** | Quick action buttons (Edit, View, Delete) | ❌ No |

<!-- Screenshot: job-table-default-columns.png — Table showing default column layout -->

### Customizing Columns

Add, remove, or reorder columns to match your workflow.

**How to customize:**

<Steps>
  <Step title="Click Columns Button">
    At the top-right of the table, click the **Columns** button (looks like a column icon).
  </Step>
  
  <Step title="Show/Hide Columns">
    Check or uncheck columns to show or hide them:
    
    **Available columns:**
    - Job Number
    - Client
    - Description
    - Status
    - Priority
    - Assigned To
    - Created Date
    - Next Visit Date
    - Last Visit Date
    - Job Type (One-Time / Recurring)
    - Tags
    - Subtotal
    - Tax
    - Total Value
    - Outstanding Balance
    - Custom Fields (if configured)
  </Step>
  
  <Step title="Reorder Columns">
    Drag column names up or down to reorder them.
  </Step>
  
  <Step title="Save">
    Click **Apply**. Your column layout is saved per user.
  </Step>
</Steps>

</Tip>

## Filtering Jobs

Find exactly the jobs you need with powerful filters.

### Quick Filters (Top Bar)

<Tabs>

    Click a status chip to filter:
    
    - **All** (default)
    - **Draft**
    - **Scheduled**
    - **Unscheduled**
    - **In Progress**
    - **Completed**
    - **Invoiced**
    - **Paid**
    - **Cancelled**
    
    The number next to each status shows how many jobs match.
    
    <!-- Screenshot: status-filter-chips.png — Status filter chips with counts -->
  </Tab>
  
  <Tab title="By Assigned Tech">
    Click **Assigned To** dropdown to filter by tech:
    
    - All team members
    - Unassigned

    Admins and Managers see all jobs. Techs see only their assigned jobs.
    
    <!-- Screenshot: assigned-filter-dropdown.png — Tech filter dropdown -->
  </Tab>
  
  <Tab title="By Date Range">
    Click **Date Range** to filter by visit date:
    
    **Presets:**
    - Today
    - This Week
    - This Month
    - Next 7 Days
    - Next 30 Days
    - Custom Range (pick start and end dates)
    
    **Note:** Jobs without scheduled visits use creation date instead.
    
    <!-- Screenshot: date-range-picker.png — Date filter with presets and calendar -->
  </Tab>
  
  <Tab title="By Priority">
    Filter by urgency level:
    
    - All Priorities
    - Low
    - Medium
    - High
    - Urgent
    
    <!-- Screenshot: priority-filter.png — Priority dropdown with color-coded options -->
  </Tab>
</Tabs>

### Advanced Filters

For more complex filtering, click **Advanced Filters** (or press `Cmd/Ctrl + F`).

<AccordionGroup>
  <Accordion title="Filter by Client">
    **Use cases:**
    - View all jobs for a specific client
    - Filter by multiple clients (bulk operations)
    
    **How:**
    1. Click **Client** filter
    2. Start typing client name
    3. Select one or more clients
  </Accordion>
  
  <Accordion title="Filter by Job Type">
    **Options:**
    - One-Time
    - Recurring
    
    **Use cases:**
    - View all maintenance contracts (recurring)
    - View one-off projects only
  </Accordion>
  
  <Accordion title="Filter by Tags">
    **Use cases:**
    - View all "Plumbing" jobs
    - View all "Emergency" jobs
    - View jobs from a specific source ("Google Ad")
    
    **How:**
    1. Click **Tags** filter
    2. Select one or more tags
    3. Choose match type:
       - **Any** (job has at least one selected tag)
       - **All** (job has all selected tags)
  </Accordion>
  
  <Accordion title="Filter by Value">
    **Use cases:**
    - Find high-value jobs (over $5,000)
    - Find small jobs (under $500)
    
    **How:**
    1. Click **Value** filter
    2. Enter min and/or max value
    3. Jobs outside the range are hidden
  </Accordion>
  
  <Accordion title="Filter by Custom Fields">
    If you've added custom fields to jobs (e.g., "Job Source", "Permit Number"), you can filter by them.
    
    <Info>Learn more: [Job Custom Fields](/features/jobs/job-custom-fields)</Info>
  </Accordion>
  
  <Accordion title="Combine Multiple Filters">
    All filters work together (AND logic).
    
    **Example:**
    - Status: In Progress
    - Assigned: John
    - Priority: Urgent
    
    **Result:** Only urgent jobs assigned to John that are currently in progress.
  </Accordion>
</AccordionGroup>

<!-- Screenshot: advanced-filter-panel.png — Side panel showing all advanced filter options -->

<Tip>
  **Pro tip:** Once you've configured filters, save them as a **Saved View** (below) for quick access later.

## Sorting Jobs

Click any column header to sort by that column.

**Click once:** Sort ascending (A→Z, 1→9, oldest→newest)
**Click twice:** Sort descending (Z→A, 9→1, newest→oldest)
**Click three times:** Clear sort (back to default)

**Default sort:** Most recent first (by created date)

## Grouping Jobs

Group jobs by a field to see them organized into collapsible sections.

### How to Group

<Steps>
  <Step title="Click Group By Button">
    At the top of the table, click **Group By**.
  </Step>
  
  <Step title="Select a Field">
    **Available grouping options:**
    - Status
    - Assigned To
    - Priority
    - Job Type
    - Tags
    - Client
    - Custom Fields
  </Step>
  
  <Step title="View Grouped Results">
    Jobs are now organized into collapsible sections.
    
    Each section shows:
    - **Group name** (e.g., "Scheduled", "In Progress")
    - **Count** (how many jobs in this group)
    - **Total value** (sum of all jobs in group)
  </Step>
</Steps>

<!-- Screenshot: grouped-by-status.png — Job list grouped by status with collapsible sections -->

### Example: Group by Assigned Tech

When grouped by assigned tech, you see:

```
John Smith (12 jobs, $15,400 total)
├── JOB-1234 | ABC Plumbing | $1,200
├── JOB-1235 | XYZ Electric | $900
└── ...

Jane Doe (8 jobs, $9,800 total)
├── JOB-1240 | Smith HVAC | $2,500
├── JOB-1241 | Jones Plumbing | $1,100
└── ...

Unassigned (5 jobs, $4,200 total)
└── ...
```

<Tip>
  **Pro tip:** Grouping is great for workload balancing. Quickly see who has the most jobs or highest total value.
</Tip>

---

### Multi-Level Grouping

Group by two fields at once for deeper organization.

**Example:** Group by Status → then by Assigned Tech

```

└── Mike Johnson (7 jobs)

Scheduled (18 jobs)
├── John Smith (6 jobs)
├── Jane Doe (5 jobs)
```

## Saved Views

Save your filter + sort + group + column configurations as reusable views.

### Creating a Saved View

<Steps>
  <Step title="Configure Your Ideal View">
    Set up filters, sorting, grouping, and columns exactly how you want them.
    
    **Example:**
    - Filter: Status = In Progress
    - Assigned: John
    - Sort: By date (oldest first)
    - Columns: Job #, Client, Description, Date, Value
  </Step>
  
  <Step title="Click Save View">
    At the top-right, click **Save View** (or **⋮ More** → **Save Current View**).
  </Step>
  
  <Step title="Name Your View">
    Give it a descriptive name (e.g., "John's In-Progress Jobs").
    
    **Optional:** Set visibility:
    - **Private** (only you can see it)
    - **Shared** (everyone on your team can use it)
  </Step>
  
  <Step title="Save">
    Click **Save**. Your view is now available in the view selector.
  </Step>
</Steps>

<!-- Screenshot: save-view-modal.png — Save view dialog with name and visibility options -->

---

### Using Saved Views

Access saved views from the **View** dropdown at the top of the page.

**Default views:**
- **All Jobs** (no filters)
- **My Jobs** (assigned to me)
- **Scheduled** (has visit scheduled)
- **In Progress** (currently being worked on)
- **Completed** (finished but not invoiced)
- **Overdue** (past scheduled date but not completed)

**Your custom views** appear below the defaults.

<!-- Screenshot: view-dropdown.png — View selector with default and custom views -->

<Tip>
  **Pro tip:** Create views for common workflows:
  - "Emergency Jobs" (Priority: Urgent, Status: Unscheduled)
  - "Ready to Invoice" (Status: Completed, no invoice)
  - "High-Value Jobs" (Value > $5,000)
</Tip>

<AccordionGroup>
  <Accordion title="Edit a Saved View">

## Troubleshooting

### I can't see all jobs — only my assigned ones

**Solution:** This is correct behavior for **Technician** role. If you need to see all jobs, ask an Admin to change your role to **Manager** or **Admin**.

---

### Jobs aren't showing even though I know they exist

**Possible causes:**
1. **Active filter:** Check if you have filters applied (status, assigned, date, etc.). Click **Clear Filters**.
2. **Wrong saved view:** Check the view dropdown — you might be in "My Jobs" instead of "All Jobs".
3. **Search active:** Clear the search bar.
4. **Date range filter:** Make sure your date range includes the jobs you're looking for.

---

### Export is taking forever

**Solution:** Exports with 500+ jobs can take 1–2 minutes. If it's been more than 5 minutes, refresh the page and try again. You'll get an email with a download link.

---

### Kanban columns are empty

**Solution:** Kanban view requires jobs to have statuses. If you have only Draft jobs, they'll all be in the first column. Change job statuses or switch to Table view.

---

### I deleted a saved view by accident

**Solution:** Contact support. Admins can restore deleted views within 30 days.
