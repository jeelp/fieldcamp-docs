## Overview

Job logs are a comprehensive activity log for each job, tracking all changes, status updates, assignments, communications, and actions throughout the job lifecycle. Unlike visit logs (which track time and activities during a single visit), job logs capture the entire job history from creation to completion.

**What job logs track:**
- Job creation and modifications
- Status changes (Draft → Scheduled → Completed → Invoiced)
- Team assignments and reassignments
- Item additions and removals
- Price changes
- Visit scheduling and cancellations
- Communications (calls, emails, SMS)
- File and photo uploads
- Notes added
- Invoice generation
- Payment receipts

  <img src="/images/jobs/job-logs-timeline.png" alt="Job logs timeline view showing all job activities" />
</Frame>

<Info>
**Job Logs vs Visit Logs:** Visit logs track time spent, mileage, and work performed during a specific visit. Job logs track the complete job history across all visits and administrative actions.
</Info>

## Job Log Types

### System Logs (Automatic)

Automatically generated by FieldCamp when actions occur.

**Examples:**
- `Job created by Sarah Johnson on 2/26/2026 at 10:15 AM`
- `Status changed from Draft to Scheduled by John Smith`
- `Item added: AC Filter (qty: 2, price: $45.00)`
- `Tech assigned: Mike Davis`
- `Visit scheduled for 3/1/2026 at 2:00 PM`
- `Invoice #1234 created`
- `Payment received: $850.00 (Credit Card)`

<Frame>
  <img src="/images/jobs/system-log-entries.png" alt="System-generated log entries" />
</Frame>

### Manual Logs (User-Added)

Team members manually add log entries to document actions or decisions.

**Examples:**
- `Called customer to confirm appointment - confirmed for Tuesday 2pm`
- `Customer requested price match - manager approved 10% discount`
- `Parts ordered - ETA 3/5/2026`
- `Permit approved by city - BLD-2026-0423`
- `Customer escalation - manager called customer, resolved issue`

**How to add manual logs:**
1. Open job → **Activity** or **Timeline** tab
2. Click **+ Add Log Entry**
3. Select log type (Call, Email, Note, Action)
4. Enter description
5. Save

<Frame>
  <img src="/images/jobs/add-manual-log-entry.png" alt="Add manual log entry modal" />
</Frame>

### Communication Logs

Track all customer communications (calls, emails, SMS).

**Examples:**
- `Email sent to customer: "Visit confirmed for 3/1/2026"`
- `SMS sent to customer: "Tech is on the way"`
- `Inbound call from customer (5:32 duration)`
- `Voicemail left for customer`

**Automatic tracking:**
- Emails sent from FieldCamp → auto-logged
- SMS sent from FieldCamp → auto-logged
- Phone calls (if VoIP integration enabled) → auto-logged

### Change Logs (Audit Trail)

Track all field changes with before/after values.

**Examples:**
- `Price changed from $450.00 to $500.00 by Sarah Johnson`
- `Status changed from In Progress to Completed by Mike Davis`
- `Address changed from "123 Oak St" to "125 Oak St" by John Smith`
- `Assigned tech changed from John to Mike by Sarah`

**Audit data:**
- Who made the change
- When the change was made
- Old value → New value
- IP address (for security auditing)

<Warning>
**Compliance:** Change logs are immutable and cannot be deleted or edited. Required for SOC 2 compliance and financial audits.
</Warning>

## Viewing Job Logs

### Timeline View (Default)

Chronological view of all job activities, newest first.

**How to access:**
1. Open any job
2. Go to **Timeline** tab (or **Activity** tab in some views)
3. Scroll to see all logs

**Timeline features:**
- Color-coded by log type (blue = system, green = communication, orange = change)
- Expandable entries (click to see full details)
- Time grouping (Today, Yesterday, This Week, Older)
- Search and filter

<Frame>
  <img src="/images/jobs/timeline-view-grouped.png" alt="Timeline view with time grouping" />
</Frame>

### List View

Table format showing all log entries.

**Columns:**
- Date & Time
- User (who performed the action)
- Action Type
- Description
- Details (expandable)

**How to access:**
1. Open job → **Logs** tab
2. Toggle view to **List**

<Frame>
  <img src="/images/jobs/logs-list-view.png" alt="Job logs in list view" />
</Frame>

### Audit View (Admin Only)

Detailed audit view showing all changes with before/after values.

**How to access:**
1. Open job → **Logs** tab
2. Click **Audit Mode** (admin only)
3. View all field changes with diffs

**Diff display:**
```
Status: Draft → Scheduled
Price: $450.00 → $500.00
Assigned: [none] → John Smith
```

<Info>
**Admin access only:** Audit view is restricted to admins and managers to prevent exposing sensitive internal data (e.g., who changed pricing).
</Info>

## Filtering & Searching Logs

### Filter by Log Type

Show only specific types of logs:

**Filter options:**
- All Logs
- System Logs
- Manual Logs
- Communication Logs (calls, emails, SMS)
- Change Logs (field edits)
- Payment Logs
- File Uploads

**How to filter:**
1. Open job → **Timeline** or **Logs** tab
2. Click **Filter** button
3. Select log types to show
4. Apply filter

<Frame>
  <img src="/images/jobs/log-filter-panel.png" alt="Log filter panel" />
</Frame>

### Filter by User

Show only logs from a specific team member:

1. Click **Filter → User**
2. Select user name
3. View only logs created/triggered by that user

**Use case:**
- QA review: Check all actions taken by a new tech
- Performance review: See all jobs handled by a specific tech

### Filter by Date Range

Show logs from a specific time period:

1. Click **Filter → Date Range**
2. Select:
   - Today
   - Last 7 days
   - Last 30 days
   - Custom range
3. Apply filter

### Search Logs

Full-text search across all log descriptions:

1. Type in search box (top of Logs tab)
2. Search matches description text
3. Results highlight matching keywords

**Example searches:**
- `discount` - Find all logs mentioning discounts
- `invoice` - Find invoice-related logs
- `customer called` - Find phone call logs
- `price change` - Find pricing updates

## Log Templates

Save time with pre-written log templates for common actions.

### Using Templates

1. Click **+ Add Log Entry**
2. Click **Templates** button
3. Select a template
4. Edit if needed (fill in placeholders)
5. Save

**Common templates:**
- "Called customer to confirm appointment"
- "Parts ordered - ETA [DATE]"
- "Customer requested price match - approved/denied"
- "Permit approved - #[PERMIT_NUMBER]"
- "Escalation to manager - [REASON]"

<Frame>
  <img src="/images/jobs/log-templates-picker.png" alt="Log templates dropdown" />
</Frame>

### Creating Custom Templates

1. Go to **Settings → Jobs → Log Templates**
2. Click **+ New Template**
3. Enter template text (use `[PLACEHOLDERS]` for dynamic fields)
4. Select log type (Call, Email, Note, Action)
5. Save

**Example template:**
```
Customer called about [ISSUE]. Resolved by [RESOLUTION]. Follow-up scheduled for [DATE].
```

<Tip>
**Placeholders:** Use `[DATE]`, `[TIME]`, `[TECH]`, `[CUSTOMER]`, and other placeholders that auto-fill when the template is used.
</Tip>

See [Job Log Templates](/features/jobs/job-logs-templates) for full documentation.

## Exporting Job Logs

### Export Single Job Logs

Export logs for one job:

1. Open job → **Logs** tab
2. Click **Export**
3. Select format:
   - PDF (formatted timeline)
   - CSV (tabular data)
   - JSON (structured data)
4. Download file

**PDF export use cases:**
- Attach to invoice (show work timeline)
- Customer audit request
- Internal quality review

**CSV export use cases:**
- Import into Excel for analysis
- Compliance reporting
- Data migration

### Bulk Export Logs

Export logs for multiple jobs:

1. Go to **Jobs → List View**
2. Select jobs (checkboxes)
3. Click **Bulk Actions → Export Logs**
4. Select format (PDF, CSV, JSON)
5. Download .zip file (one file per job)

## Log Notifications

Get notified when specific log events occur.

### Notification Rules

Set up rules to trigger notifications:

1. Go to **Settings → Notifications → Job Logs**
2. Click **+ Add Rule**
3. Configure:
   - **Trigger:** When [log type] is created
   - **Condition:** Where [field] matches [value]
   - **Action:** Send notification to [user/role]
4. Save rule

**Example rules:**
- When "Price changed" log is created → Notify manager
- When "Escalation" log is created → Notify admin
- When "Payment received" log is created → Notify accounting team
- When manual log contains "urgent" → Notify dispatcher

<Frame>
  <img src="/images/jobs/log-notification-rule.png" alt="Log notification rule configuration" />
</Frame>

### Real-Time Notifications

Subscribe to real-time log updates for specific jobs:

1. Open job → **Logs** tab
2. Click **Subscribe** (bell icon)
3. Receive notifications for all new logs on this job

**Notification delivery:**
- In-app notification (bell icon)
- Email digest (hourly or daily)
- Mobile push notification (if mobile app installed)

**Unsubscribe:**
Click **Unsubscribe** to stop notifications for that job.

## Best Practices

### ✅ Do This

- **Add manual logs for key decisions** (price approvals, escalations, customer requests)
- **Use templates** to standardize log entries (ensures consistency)
- **Log all customer communications** (even quick phone calls)
- **Review logs before invoicing** to ensure all work is documented
- **Use logs for QA** (review job logs weekly to catch issues)
- **Export logs for compliance** (monthly/quarterly exports for audits)

### ❌ Avoid This

- **Don't skip manual logs** (if it's not logged, it didn't happen)
- **Don't use vague descriptions** ("Talked to customer" is less useful than "Customer confirmed 2pm appointment")
- **Don't log sensitive data** (credit cards, SSN, passwords)
- **Don't rely only on system logs** (system logs miss context and decisions)
- **Don't delete logs** (they're immutable for a reason)

## Troubleshooting

### Logs Not Appearing

**Possible causes:**
- Logs filtered out (check filter settings)
- User doesn't have permission to view logs
- Sync delay (on mobile app)

**Solution:**
1. Clear all filters (click "Reset Filters")
2. Check user role permissions (Settings → Roles)
3. On mobile: Pull to refresh to sync logs

### Communication Logs Missing

**Cause:**
VoIP integration not enabled or email/SMS not sent from FieldCamp.

**Solution:**
- Enable VoIP integration (Settings → Integrations → Phone)
- Send emails from FieldCamp (not external email client) to auto-log
- Manually log external communications (if email sent from Outlook, add manual log)

### Can't Export Logs

**Cause:**
User role doesn't have export permission.

**Solution:**
Ask an admin or manager to export logs for you.

## API Integration

Retrieve and create logs programmatically.

### Get Job Logs

```bash
curl -X GET https://api.fieldcamp.ai/v1/jobs/{job_id}/logs \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**
```json
{
  "logs": [
    {
      "log_id": "log_123",
      "job_id": "job_456",
      "type": "status_change",
      "description": "Status changed from Draft to Scheduled",
      "user": {
        "id": "user_789",
        "name": "John Smith"
      },
      "timestamp": "2026-02-26T10:15:00Z",
      "metadata": {
        "old_value": "draft",
        "new_value": "scheduled"
      }
    },
    {
      "log_id": "log_124",
      "job_id": "job_456",
      "type": "manual",
      "description": "Called customer to confirm appointment",
      "user": {
        "id": "user_789",
        "name": "John Smith"
      },
      "timestamp": "2026-02-26T11:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 50,
    "total": 127
  }
}
```

### Create Manual Log

```bash
curl -X POST https://api.fieldcamp.ai/v1/jobs/{job_id}/logs \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "manual",
    "description": "Customer requested price match - manager approved 10% discount",
    "user_id": "user_789"
  }'
```

### Filter Logs

```bash
curl -X GET "https://api.fieldcamp.ai/v1/jobs/{job_id}/logs?type=communication&date_from=2026-02-01&date_to=2026-02-28" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

See full API documentation at [Job Logs API Reference](/api-reference/job-logs).
</Tip>

## Integration with Other Modules

### Job Logs ↔ Visit Logs

**Relationship:**
- Visit logs are event-based (time tracking, work performed during a visit)
- Job logs reference visit logs when visits are scheduled/completed

**Example:**
```
Job Log: "Visit scheduled for 3/1/2026 at 2:00 PM" (links to visit)
Visit Log: "Tech arrived at 2:05 PM, work started at 2:10 PM" (time tracking)
Job Log: "Visit completed on 3/1/2026" (references visit completion)
```

### Job Logs ↔ Invoicing

**Automatic logs:**
- Invoice created → `Invoice #1234 generated`
- Invoice sent → `Invoice #1234 sent to customer via email`
- Payment received → `Payment received: $850.00 (Credit Card)`

**Manual logs:**
- Add notes about payment plans, disputes, or approvals

### Job Logs ↔ Workflow Automation

Use job logs as workflow triggers:

**Example workflows:**
- When log type = "Escalation" → Send email to manager
- When log description contains "urgent" → Create high-priority task
- When log type = "Payment Received" → Mark job as "Paid" and send thank-you email

## FAQ

<AccordionGroup>
  <Accordion title="Can I edit or delete a log entry?">
    No. Job logs are immutable (cannot be edited or deleted) to maintain audit integrity. If you made a mistake, add a correcting log entry.
  </Accordion>

  <Accordion title="How long are job logs retained?">
    Job logs are retained indefinitely by default. Enterprise plans can configure custom retention policies (e.g., delete logs after 7 years for GDPR compliance).
  </Accordion>

  <Accordion title="Can customers see job logs?">
    No. Job logs are internal-only. Customers can see customer-facing notes (in the Notes feature) but not the full job log timeline.
  </Accordion>

  <Accordion title="Do job logs count toward my storage limit?">
    No. Job logs are metadata (small text entries) and do not count toward your storage limit. Only files and photos count.
  </Accordion>

  <Accordion title="Can I bulk-import historical logs?">
    Yes. Use the API or CSV import to bulk-create manual logs. System logs cannot be retroactively created.
  </Accordion>

  <Accordion title="What's the difference between job logs and job notes?">
    **Job notes** are user-written comments (can be customer-facing or internal). **Job logs** are a complete activity trail (system + manual, immutable, audit-focused).
  </Accordion>
</AccordionGroup>
